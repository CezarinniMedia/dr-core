## Key Takeaways for DR OPS

### 1. Color System Recommendation
The overwhelming preference is **dark mode** (65% of references) with **green/teal as the primary accent** (39%). This aligns perfectly with DR OPS's existing dark theme. Recommended palette:
- **Background:** #0f1117 to #1a1a22 (very dark charcoal with slight blue/green tint)
- **Card surfaces:** #1e2028 to #252830 (elevated dark cards)
- **Primary accent:** #00cc88 to #00e5a0 (teal-green, for positive actions, success, CTAs)
- **Secondary accent:** #4488ff (blue, for informational elements, links)
- **Danger/alert:** #ff5555 (red-coral, for deletions, negative metrics)
- **Neutral text:** #ffffff (headings), #b0b8c8 (body), #6a7080 (muted labels)

### 2. Glass Effect Usage
Glassmorphism appears in 32% of references — but should be used sparingly in DR OPS:
- **DO use:** For floating modals, tooltips, popovers (Images 02, 07, 11, 13, 15)
- **DO NOT use:** For main content areas or data-dense tables (performance concern with 14k+ records)
- **Subtle approach:** Slight background blur (4-8px) + semi-transparent background + thin border

### 3. Dashboard Layout Pattern
Images 30 and 31 are the most directly applicable:
- **Sidebar navigation** (collapsible, icon-only when collapsed)
- **KPI row at top** (3-5 metric cards with large values + change indicators)
- **Main chart area** (area/line chart for trends, configurable time range)
- **Secondary content** (data table, activity feed, or secondary charts)
- **Optional:** AI chat panel overlay (Image 30)

### 4. Component Patterns to Adopt
- **Pill-shaped toggle selectors** for time ranges (Monthly/Yearly, 1W/1M/6M/1Y)
